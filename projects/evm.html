This project aims to develop a predictive model to classify individuals as good or bad credit risks using the Statlog (German Credit Data) dataset. By leveraging attributes such as credit history, duration of credit, and residence. The main objectives of this project was to create a reproducible and trustworthy data flow and to establish a consistent predictive model.

In the financial sector, accurately assessing credit risk is crucial for lending decisions. The Statlog (German Credit Data) dataset provides a comprehensive set of attributes that can help in classifying individuals as good or bad credit risks.
We utilized the data to develop a predictive model that classifies credit risks effectively, focusing on ensuring reproducibility and trustworthiness in the data flow. We analyzed key attributes such as credit history, duration of credit, and residence. Various modeling techniques were implemented, with a primary focus on Logistic Regression and Random Forest models. Throughout the process, we emphasized in creating a structured data flow to ensure that the model could be consistently reproduced and validated.
As a result of our efforts, we successfully developed a predictive model that fairly accurately classifies credit risks while maintaining a reproducible and trustworthy data flow, providing valuable insights for financial institutions in their lending practices.

Tools Used

  • Python, scikit-learn, pandas, NumPy for data analysis and model training
  • Docker, Conda for ensuring reproducibility
  • Quartoe for creating dynamic reports
  • GNU Make for automation and efficiency
  • GitHub Actions for CI/CD
  • pytest for testing to ensure functionality and reliability
  • PyPI for packing project and increasing reproducibility

Analysis Summary

Initially, a dummy model served as a baseline, followed by logistic regression to interpret the influence of individual features on credit risk. To address logistic regression's limitations in capturing complex, non-linear relationships, a random forest model was introduced, enhancing predictive performance through multiple decision trees. The dataset was split 80-20 for training and testing, with categorical variables one-hot encoded and continuous variables standardized. Key evaluation metrics—precision, recall, accuracy, and F1 score—were selected to mitigate the real-world consequences of false positives and negatives, ensuring both statistical robustness and fairness in credit risk assessment.Our best-performing model is a Random Forest model which gave us a test score of 0.754 on unseen data, a decent result compared to the dummy model's score of 0.699.
